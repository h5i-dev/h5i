# h5i

**Elevating Git for the AI Era. The 🙌high-five🙌 between Humans and AI.**

**h5i** (pronounced *high-five*) is a high-performance version control engine designed for the era of **Software 3.0**. While traditional Git tracks code as a 1-dimensional stream of text , **h5i** operates across **five distinct dimensions of software truth**, allowing humans and AI agents to collaborate with unprecedented semantic clarity.

---

## 🌟 Core Technologies

### ⚡ Rust-Powered Core

Engineered for sub-second latency in massive monorepos. **h5i** leverages Rust's safety and speed to handle the high-frequency commit volume generated by AI agents.

### 🔄 Seamless Git Interop

**h5i** is a "porcelain sidecar". It populates your standard `.git` directory while maintaining its 5D metadata in a hidden `.h5i` folder. Your existing GitHub Actions and IDEs will work out of the box.

### 🧩 Plug-and-Play AST

AST support is **optional and extensible**. Use our standard JSON/S-expression interface to plug in any parser (Python, Rust, TypeScript). If a parser isn't available, **h5i** gracefully falls back to 1D line diffing.

---

## 🚀 Commands

| Command | Dimension | Description |
| --- | --- | --- |
| `h5i init` | Setup | Initializes the 5D sidecar in a Git repo. |
| `h5i commit` | Intent/Quality | Commits code with AI metadata and test snapshots. |
| `h5i sync` | Temporal | Merges local CRDT updates from agents/humans. |
| `h5i log --ai` | Intent | Displays the prompt-history and model-provenance. |
| `h5i blame --ast` | Structural | Tracks logic ownership across renames and moves. |
| `h5i verify` | Empirical | Checks if the current state meets quality/coverage gates. |
